Responding to Comments from a VP in an Excel Model

← Back to Excel Examples

“Analysts are regularly expected to revise Excel models in response to feedback from senior team members. These comments might appear as in-line notes, comment balloons, or instructions in Slack. How you interpret, implement, and communicate these changes reflects your professionalism and directly impacts turnaround time, clarity, and trust.”

Step 1: Read Each Comment Carefully

Before doing anything, read the comment twice. Ask:

  • Is it a question or a directive?
  • What exactly is being requested?

Example:
VP writes: “Should we increase the exit cap to 5.5% based on the new comp set?”
This is not a direct instruction — it’s a prompt to consider and verify.

Step 2: Make the Requested Change in the Model

Apply the change precisely and confirm that formulas update downstream.

Example:
You update the exit cap from 5.0% to 5.5% in the assumptions tab and verify it flows through to the valuation.

Step 3: Leave a Brief Reply or Note Near the Change

Use a reply comment or nearby note to confirm what you did.

Example:
“Updated exit cap to 5.5% as discussed — see changes reflected in row 130.”

Step 4: Ask for Clarification if Needed

If you’re unsure, flag it with a note or message the reviewer.

Example:
“Used 5.5% based on IR team comp set — happy to revert if needed.”

Step 5: Mark Comments as Resolved

  • Click “Resolve” (in threaded comments)
  • Or delete the comment after you’ve responded

Avoid leaving resolved comments visible.

Step 6: Summarize Updates When Sharing the File

When you send the file back, provide a bullet summary:

Changes made to Investor Model v3.1:
• Updated exit cap to 5.5% (Tab 2, Row 130)
• Corrected IRR calc for updated NOI
• Recolored historicals for consistency
